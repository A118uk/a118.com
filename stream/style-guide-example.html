Feature list — bullet rule

Rule: when a list has exactly 5 items, use the cycling-colour square so each item carries one stripe colour in order. For any other count, use the stacked stripe dashes so every bullet carries the full palette.
5 items · cycling colour square

Orange — first

Each item carries one colour from the stripe in order, giving the list a natural rhythm.

Amber — second

Works best when the list is intentionally exactly five things — e.g. the five palette colours.

Olive — third

Colours never feel arbitrary because they map back to the identity stripe.

Sage — fourth

Rounded square form keeps it consistent with the card and badge language.

Mauve — fifth

The cycle closes naturally at five, matching the palette length exactly.

3 items · stacked stripe dashes

Full palette per bullet

Every bullet carries the five stripe colours stacked as 2px dashes, so no item feels arbitrary.

Length-agnostic

Looks right at 2 items, 3, 4, 6, or 20 — the stripe logic doesn't depend on list length.

Safe default

Use this whenever the list isn't exactly five items.

7 items · stacked stripe dashes

Item one

Longer lists stay visually consistent; no colour fights for attention.

Item two

Each dashes bullet is self-contained — it doesn't rely on position in the list.

Item three

Dashes read as a decorative micro-logo rather than an indicator.

Item four

Keeps dense list sections restrained and legible.

Item five

No special treatment for the fifth item — this is the whole point.

Item six

Pattern scales to any length without the cycle getting weird.

Item seven

Safe to use for anything that isn't exactly five.
